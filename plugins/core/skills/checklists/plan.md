# Feature Planning Checklist

## Step 1: Clarify Requirements
Identify unclear requirements that need user input:
- Business logic ambiguities
- UX/workflow preferences
- Priority tradeoffs
- Data ownership/access questions

Technical decisions (tools, libraries, patterns) can be made independently.

## Step 2: High-Level Design
- Identify all components/modules needed
- Define data models and relationships
- List dependencies and external integrations
- Note potential risks or challenges

## Step 3: Implementation Plan
- Break down into ordered tasks
- Estimate complexity (simple / medium / complex)
- Identify what can be parallelized
- Note testing requirements

## Step 4: Feature Spec
Create specification at `docs/superpowers/specs/[feature-name].md` with:
- Overview
- Requirements (functional & non-functional)
- Data model
- API/Interface design
- Implementation tasks
- Open questions
